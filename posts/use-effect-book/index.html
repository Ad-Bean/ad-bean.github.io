React basics: explaining the useEffect hook

钩子 Hooks 是一种非常好用的工具，帮助你存储和管理 React 组件中的状态。

其中 useEffect 钩子提供了一种非常简单的方法，用于展示副作用 side effects

useEffect 钩子基本替代了 React 旧版本中的 class 组件的生命周期，比如 componentDidMount, componentDidupdate 和 componentWillUnmount.

你可以将 useEffect 当成一个可以带有参数的唤醒者 an invoker that takes arguments，下面有几个例子：

Using the useEffect hook

为了使用这个钩子，需要先从 React 中导入该函数：

import React, { useEffect } from 'react';

然后就可以使用 useEffect 钩子了，可以使用该钩子来改变网页标题 document.tittle

useEffect(() => {
    console.log('effect called');
    document.tittle = `#${count} clicks`;
})

在程序运行过程中，通过使用 console.log('effect called') 来观察发生的事件：

  • Load: effect called
  • Click “Add one”, effect called
  • Click “Add one”, effect called
  • Click another state, effect called

可以看到，每当事情 (DOM 或渲染) 发生改变时，useEffect 钩子就会触发。

Only run useEffect on load

但如果我们只想要在加载时触发 useEffect 且之后不会再触发，比如初始化一些数据时：

useEffect(() => {
    loadExternalData();
}, []);

此时就只会触发一次 useEffect 钩子，为什么呢？

因为这里使用了 useEffect(callback, deps) 的第二个参数，依赖项。 当 deps 传入一个空数组时，useEffect() 就只会在加载的时候触发一次，因为它不依赖任何状态。

Running useEffect only for specific changes

大多数情况下，我们会希望 useEffect 在我们的变量发生改变时触发，比如计数器。

同样地，我们将计数器变量传入 useEffect 的第二个参数依赖项：

useEffect(() => {
    console.log('effect called');
    document.tittle = `#${count} clicks`;
}, [count]);

现在，事件发生的顺序为：

  • Load: effect called;
  • Click “Add one”, effect called
  • Click “Add one”, effect called
  • Click another state, effect NOT called

当然，依赖项并不一定只有一个，你可以传入多个依赖项：

useEffect(() => {
    document.tittle = `#${count} clicks`;
}, [count, state2, state3]);

The unmount of useEffect

useEffect 最 COOL 的地方在于，这个钩子也有 unmount state 即卸载状态

只需要在 useEffect(callback, deps) 的回调函数中使用 return 即可：

useEffect(() => {
    // do some effect
    return () => {
        // clean up
    };
}, [state]);

需要记住的是，当状态 state 每次发生改变时，不仅会触发钩子产生副作用 effect，也会进行清除工作即 cleanup。

为了展示这个过程，我们重述上述的流程为：

  • Load: effect called
  • Click “Add one”, cleanup called, effect called
  • Click “Add one”, cleanup called, effect called
  • Click another state, effect NOT called
  • Unmount app, cleanup called

清除工作 cleanup 可以帮助你清理一些副作用，正如其名，你可以：

  • 清理 Fetch 请求
  • 清理 Timers 计时器
  • 清理 Web sockets 连接
  • 清理 Debounce/Throttle 防抖/节流

举个例子，在使用 Websocket 时，只需要在组件加载时触发一次开启连接，组件关闭或离开时关闭连接：

function App() {
  const [socketMessage, setSocketMessage] = useState('');

  useEffect(() => {
    const socket = new WebSocket('wss://localhost:8000');
    socket.onmessage = (event) => {
      setSocketMessage(JSON.parse(event.data));
    };

    return () => {
      socket.close();
    };
  }, [socketMessage]);
}

此时我们可以很好地在离开组件后，关闭不再使用的 Socket 连接。

Conclusion

总地来说，在如今函数组件作为主流的情况下，useEffect(callback, deps) 可以帮助我们在不同生命周期下执行一些副作用，比如当组件加载时获取数据、开启 WS 连接，并且通过依赖项进行按需触发。在回调函数中使用 return ()=> {} 进行清理，帮助我们关闭不再需要的 WS 连接，取消定时器等等。
