项目已经通过编译得到了可执行文件，通过解压 Build.zip 后可以直接运行 TicTacToe.exe 进行井字棋游戏。

项目代码与场景在 Assets/ 文件夹下，Scences 为场景，Scripts 为脚本文件，如果需要加载场景与脚本到 Unity 中，可以新建 Unity 项目后将 Assets 文件夹拖入 Unity 中，双击 Home 场景后，即可运行游戏。如果遇到某个场景无法加载的问题，需要到 Unity -> File -> Build Settings 或者直接使用 ctrl + shift + B 打开编译环境，将所有场景逐一拖入加载，不需要点击 Build 或 Build and Run 即可关闭：

三个模式分别为：与 AI 对战，双人游戏，观看 AI 对战，单击任意选项即可进入游戏：

HomePage

项目的主要场景是 Home 即主页，使用 HomeComponent.cs 组件加载主页，使用 OnGUI() 渲染各个 Button 与场景。

Two Player

TwoPlayer 场景是双人游戏模式，使用 TwoPlayerComponent 组件控制，该组件继承于 GameComponent 组件，重载了其 NextRound() 函数，表示下一个棋手落棋的位置：

PlayWithAI

PlayWithAI 场景是玩家与 AI 对战，使用 AIComponent 组件控制，该组件同样继承于 GameComponent 组件，但为了实现 AI 棋手，需要实现一个 AI 组件 AIPlayer，每次获取棋盘和游戏的状态，对玩家的棋子进行判断从而落棋。

实际上该 AI 比较简单，如果用户的棋子快连成线，则落棋堵住用户的棋子；如果自己的棋子将要连成线，则尝试落棋连成线得到胜利；否则其他情况则随机落棋。

ECS 架构：

ECS，即 Entity-Component-System（实体-组件-系统） 的缩写，其模式遵循组合优于继承原则。游戏内的每一个基本单元都是一个实体，每个实体又由一个或多个组件构成，每个组件仅仅包含代表其特性的数据（即在组件中没有任何方法）。

在本次项目中我尽量遵循 ECS 架构，井字棋棋盘状态、游戏状态、AI 棋手等等都做成了组件，每个组件都拥有相应的属性。
