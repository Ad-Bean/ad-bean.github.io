Writing Markup With JSX

使用 JSX 编写标记语言：JSX 是 Javascript 的扩展语法，JSX 可以让你在 Javascript 文件中编写类似 HTML 的标记语言。尽管还有其他方法来编写组件，但大部分 React 开发人员更喜欢 JSX 的简洁性，而且大多数代码库都使用 JSX。

在本章，你会学到：

  • 为什么 React 要将标记语言和渲染逻辑混杂在一起
  • JSX 和 HTML 标记语言有什么不同
  • 如何使用 JSX 展示信息

JSX: Putting markup into JavaScript

JSX 将标记语言放入 Javascript 中。现在的 Web 大部分建立在 HTML / CSS / Javascript 之上，多年以来，网页开发人员一直使用 HTML 来标记内容，CSS 来添加样式，JS 来添加逻辑和动作，并且它们常常是分开的文件，网页的内容即 HTML 往往和逻辑即 Javascript 分离开来。

但是随着 Web 的发展，我们需要更多的交互功能，逻辑变得越来越重要，甚至逻辑可以决定内容。Javascript 可以掌控 HTML，这也是为什么在 React 中，渲染逻辑和标记语言可以放在一起，放在同一个组件中。

将一个按钮的渲染逻辑和标记语言放在一起，可以确保它们在每次编辑时保持同步。相反地，不相关的细节比如按钮的标记语言和侧边栏的标记语言，是相互孤立的，单独修改它们是安全的。

React 中每个组件都是一个含有许多标记语言（React 会将其渲染到浏览器）的 Javascript 函数。React 组件使用 JSX 扩展语法来表示这些标记语言。JSX 看上去很像 HTML，但它实际上更加严格，并且可以展示动态信息。理解 JSX 的最好方法就是尝试自己将一些 HTML 标记语言转换成 JSX 标记。

注意：

JSX 和 React 是两种不同的事物，你可以分别使用任意一种，也可以将其组合。

The Rules of JSX

JSX 的规则有：必须返回单个根组件，必须闭合所有标签，使用驼峰命名来表示大部分事情

1. Return a single root element

返回单一根元素：为了在组件中返回多个元素，你必须使用一个父标签来将他们围绕起来，比如你可以使用一个 <div>

更多知识：

为什么多个 JSX 标签需要被父标签包含？

尽管 JSX 扩展语法看起来很像 HTML，但实际上他被转换成了纯 Javascript 对象。一个 JS 函数不能返回两个对象，你必须将他们放入一个数组中（以前有种写法 return [<li></li><li></li>]）。这就解释了如果不将两个 JSX 标记包装到另一个标记或片段中，就不能返回两个 JSX 标记。

2. Close all the tags

关闭所有标签：JSX 要求所有标签显式地关闭，甚至像 <img> 这类自闭合（单）标签也需要 <img /> 关闭，并且必须要严格地使用闭标签，比如 <li>oranges</li>

3. camelCase all most of the things!

对几乎所有属性都使用驼峰命名：JSX 转换成 Javascript 时，用 JSX 编写的属性键值可以映射到 Javascript 对象。在你的组件中，你时常需要读写这些属性，但 Javascript 对这些属性的名字存在限制。比如，属性名不能含有破折号 - 或者保留关键字比如 class

这也是为什么在 React 中，许多 HTML 和 SVG 属性使用驼峰写法即 camelCase。比如使用 strokeWidth 而不是 stroke-width，又因为 class 是一个保留关键字，在 React 你应该使用 className，根据相应的 DOM 属性来命名

你可以在 React DOM 元素 找到相应的属性名，如果你遇到了报错，React 会将报错信息输出到浏览器控制台

陷阱：

由于历史原因，aria-* 和 data-* 带有破折号的属性依然使用 HTML 带有破折号的写法（前者用于无障碍 Accessible Rich Internet Applications，后者用于 HTML5 中的数据属性，可以通过 document.querySelector('#electriccars').dataset.indexNumber 访问属性 data-index-number="123" 的数据）

Pro-tip: Use a JSX Converter

提示，使用 JSX 转换器，将所有 HTML 标记语言中的属性转换成 JSX 可能会非常繁琐，React 团队推荐你使用在线转换器将 HTML / SVG 转换成 JSX。转换器在实践中是非常有用的，但是了解如何将其转换成 JSX 语法可以帮助你更好地编写 JSX。
