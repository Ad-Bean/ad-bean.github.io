Hugo 和 Loveit 主题

Hugo: The world’s fastest framework for building websites

Hugo is one of the most popular open-source static site generators. With its amazing speed and flexibility, Hugo makes building websites fun again.

LoveIt: A Clean, Elegant but Advanced Hugo Theme for Hugo.

为什么选择 Hugo

Hugo 号称世界上最快的构建网站工具，其速度体现在生成静态文件时，要比基于 Node.js 且有许多插件的的 Hexo 要快上好几倍。

Hugo 基于 Go 语言实现，使用自带的 Go html 模板，生成静态文件仅需几秒到十几秒，当博客的文章、主题、样式、图片越来越多、越来越复杂的时候，Hugo 的优势便显得极为明显。尽管 Hexo 使用更为广泛，且主题更加丰富，但我还是选择了 Hugo。

本文就将以 Hugo 出发，加以由 Dillon 开发的 LoveIt主题，构建一个简洁、优雅且高效 Hugo 博客。

准备工作

安装 Hugo

Linux 环境

如果你在 Linux 环境下，可以使用 Homebrew 管理工具来安装 Hugo(macOS 同理)

brew install hugo
Windows 环境

如果你使用的是 Windows，请使用 Chocolatey 或者 Scoop 包管理工具来安装 Hugo，下面将从安装 Chocolatey 开始，一步步安装 Hugo

请使用管理员权限运行 powershell，并且确保 Get-ExecutionPolicy 不是 Restricted 的。

  1. 按下键盘上的 win+x ，以管理员模式运行 powershell。
  2. 输入 Get-ExecutionPolicy 并运行，若返回 Restricted，则输入 Set-ExecutionPolicy AllSigned 或 Set-ExecutionPolicy Bypass -Scope Process。
  3. 输入下面的命令并且运行
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))
  1. 等待一会后，若无报错。则表明 Chocolatey 安装成功，输入 choco 查看可以当前版本以确保安装成功。
Chocolatey v0.10.15
Please run 'choco -?' or 'choco <command> -?' for help menu.
  1. 安装 Hugo 和 Hugo extended
choco install hugo -confirm
choco install hugo-extended -confirm

在命令行输入 hugo version 以验证 hugo 正确安装

hugo version
>>> Hugo Static Site Generator v0.74.3
  1. 为了后续发布静态页面到 github pages 或是 gitee pages，请安装 Git 和 vscode

由于 git 和 vscode 等工具的具体使用与本文相关性不大，故不在此介绍其下载和使用方式。

搭建博客

下面的步骤将基于 LoveIt 主题搭建博客，如果需要，你可以查阅 Hugo 文档了解更多，同时为了更好的理解，你可以需要阅读 Hugo 提供的 Quick Start。

创建一个新项目

使用 Hugo new 命令创建一个新的项目：

hugo new site my_blog

同时使用 vscode 打开这个新建项目所在的文件夹，当然你也可以仅用命令行操作，这一切取决于你的习惯。

安装主题

LoveIt 主题的仓库是: https://github.com/dillonzq/LoveIt.

我们在 my_blog 目录下使用 git 命令将主题仓库 clone 到本地 themes 文件夹下

git clone https://github.com/dillonzq/LoveIt.git themes/LoveIt

基础配置

TOML 是前 GitHub CEO， Tom Preston-Werner，于 2013 年创建的语言，其目标是成为一个小规模的易于使用的语义化配置文件格式。TOML 被设计为可以无二义性的转换为一个哈希表(Hash table)。TOML 是大小写敏感的，必须是 UTF-8 编码。

下面打开处于 my_blog 文件夹下的 config.toml 文件，对其进行编辑。

# 设置baseURL且使用相对URL
baseURL = "http://example.org/"
relativeURLs = true

# [en, zh-cn, fr, ...] 设置默认的语言
defaultContentLanguage = "zh-cn"
# 网站语言, 仅在这里 CN 大写
languageCode = "zh-CN"
# 是否包括中日韩文字
hasCJKLanguage = true
# 网站标题
title = "我的全新 Hugo 网站"

# 更改使用 Hugo 构建网站时使用的默认主题
theme = "LoveIt"

[params]
  # LoveIt 主题版本
  version = "0.2.X"

[menu]
  [[menu.main]]
    identifier = "posts"
    # 你可以在名称 (允许 HTML 格式) 之前添加其他信息, 例如图标
    pre = ""
    # 你可以在名称 (允许 HTML 格式) 之后添加其他信息, 例如图标
    post = ""
    name = "文章"
    url = "/posts/"
    # 当你将鼠标悬停在此菜单链接上时, 将显示的标题
    title = ""
    weight = 1
  [[menu.main]]
    identifier = "tags"
    pre = ""
    post = ""
    name = "标签"
    url = "/tags/"
    title = ""
    weight = 2
  [[menu.main]]
    identifier = "categories"
    pre = ""
    post = ""
    name = "分类"
    url = "/categories/"
    title = ""
    weight = 3

# Hugo 解析文档的配置
[markup]
  # 语法高亮设置 (https://gohugo.io/content-management/syntax-highlighting)
  [markup.highlight]
    # false 是必要的设置 (https://github.com/dillonzq/LoveIt/issues/158)
    noClasses = false

创建第一篇文章

使用 Hugo 的 new 命令在 posts 目录下创建你的第一篇 markdown 文章：

hugo new posts/my_first_post.md

默认情况下, 所有文章和页面均作为草稿创建. 如果想要渲染这些页面, 请从元数据中删除属性 draft: true, 设置属性 draft: false 或者为 hugo 命令添加 -D/–buildDrafts 参数.

使用 Tags: [“XX”, “XXX”] 和 categories: [“XXX”] 让你的文章易于分类！

在本地启动网站

hugo serve

运行上方命令后，你就可以在 http://localhost:1313 查看你所生成的页面了 由于此时是动态页面，每当你保存的时候，页面就会随着改变进行刷新。

构建网站

一切准备就绪，你可以生成你的静态页面，然后发布在 Github Pages 或是 Gitee Pages 上以供大家查看了！

hugo

该命令会生成一个 public 目录, 其中包含你网站的所有静态内容和资源。

查阅或搜索 Github Pages 和 Gitee Pages 等服务，将你的网页发布上去吧！

更多配置

或许你会觉得此时的页面还太死板，想要更多个性化的样式。

我们可以通过查阅 LoveIt 官方文档来对配置文件进行修改，达到个性化的操作！

由于 LoveIt 是由国人开发的，有中文官方文档，这也是我选择其作为主题的原因之一。

下面我就如何更换头像，网站图标，以及如何开启 KATEX 进行数学公式的编写略写几点。

更换头像、网站图标和开启数学公式

我们把需要的图片，比如 avatar.png, favicon.ico 放在 my_blog/static 目录下。

对 config.toml 文件进行修改：

[params]
    [params.math]
        # 开启数学公式
        enable = true
    [params.home]
        # 主页个人信息
        [params.home.profile]
            enable = true
            # 主页显示头像的 URL
            avatarURL = "/avatar.png"
      # LoveIt 新增 | 0.2.0 应用图标配置
    [params.app]
        # 更现代的 SVG 网站图标, 可替代旧的 .png 和 .ico 文件
        svgFavicon = "/favicon.ico"

当然，你的博客你做主，除了以上的配置以外，还有更多的个性化配置等你去探索。
