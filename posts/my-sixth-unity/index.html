项目文档

项目配置过程

将 Assets 文件夹下的文件全拖入新建项目后，点击新导入的 Scene/ 中的 Patrol Soldier 场景，开始游戏后即可打开项目。

游戏规则

  • 使用 WASD 前后左右移动人物，按住 Shift 进行奔跑，使用 Space 空格进行跳跃
  • 操作主角躲避巡逻兵，每到达一个新区域加一分，总分达到十分获得胜利，否则被巡逻兵抓住游戏失败。

项目架构

本次项目依然按照 MVC 框架实现，并且使用动作分离、订阅与发布模式传消息。

Model

  • GameModels ：游戏实体类，管理游戏当前状态，游戏得分、控制游戏加分和重置。
  • Soldier ：士兵实体类，获取巡逻士兵的区域，进行巡逻和移动。

View

  • GameGUI ：显示游戏得分，开始菜单和退出按钮。

Controller

  • AreaCollider ：区域碰撞控制器，判断玩家进入（碰撞）的区域。
  • Director ：导演类，控制场景切换。
  • GameController ：游戏控制器，获取玩家操作，进行相应的动画和场景控制。
  • GameEventManager ：高级游戏事件管理，获取碰撞事件，判断玩家进入区域或碰撞巡逻兵。
  • Map：地图控制器，生成相应的资源：围栏、巡逻兵、地面。
  • PlayerCollider：判断玩家能否跳跃的碰撞控制器。
  • SoldierCollider：判断玩家是否碰到巡逻兵的碰撞控制器。

Actions 动作管理

  • Action：基础的动作类，用于继承。
  • ActionManager：动作管理类，使用字典管理相应的动作。
  • GameActionManager：该游戏的动作管理类，用于控制巡逻兵。
  • MoveToAction：控制巡逻兵移动到相应位置。
  • TraceAction：控制巡逻兵追赶玩家。

订阅与发布模式

使用 AreaCollider 检测碰撞，当玩家进入某个区域时，利用 OnTriggerEnter() 钩子函数更新/发布玩家当前所在区域的号码，利用订阅与发布模式，在 GameController 中解耦/订阅玩家进入新区域事件。

public class AreaCollider : MonoBehaviour {
    public int area;
    
    public void OnTriggerEnter(Collider collider) {
        if (collider.gameObject.tag == "Player") {
            GameEventManager.GetInstance().PlayerEnterArea(area);
        }
    }
}

使用 SoldierCollider 检测玩家与巡逻兵碰撞，利用 OnCollisionEnter() 钩子函数更新/发布玩家是否与巡逻兵碰撞事件，利用订阅与发布模式，在 GameController 中解耦/订阅相应事件。

public class SoldierCollider : MonoBehaviour {
    public void OnCollisionEnter(Collision collision) {
        if (collision.gameObject.tag == "Player") {
            GameEventManager.GetInstance().SoldierCollideWithPlayer();
        }
    }
}

GameController 中订阅玩家进入新区域事件、订阅玩家与巡逻兵碰撞事件：

void Awake() {
    model.onRefresh += delegate {
        view.state = model.state;
        view.score = model.score;
    };
    view = gameObject.AddComponent<GameGUI>();
    actionManager = gameObject.AddComponent<GameActionManager>();
    // 
    GameEventManager.onPlayerEnterArea += OnPlayerEnterArea;
    GameEventManager.onSoldierCollideWithPlayer += OnSoldierCollideWithPlayer;
    Director.GetInstance().OnSceneWake(this);
}

// 玩家进入新区域
private void OnPlayerEnterArea(int area) {
    if (model.state != GameState.RUNNING) { return; }
    if (currentArea != area) {
        model.AddScore(1);
        soldiers[currentArea].GetComponent<Soldier>().isFollowing = false;
        soldiers[currentArea].GetComponent<Animator>().SetFloat("Speed", 0.5f, 0.1f, Time.deltaTime);
        currentArea = area;
        soldiers[currentArea].GetComponent<Soldier>().isFollowing = true; 
        actionManager.Trace(soldiers[currentArea], player);
        soldiers[currentArea].GetComponent<Animator>().SetFloat("Speed", 1f, 0.1f, Time.deltaTime);
        if (model.score == model.targetScore) {
            view.state = model.state = GameState.WIN;
            Cursor.lockState = CursorLockMode.Confined;
            playerAnimator.Play("Movement");
            soldiers[currentArea].GetComponent<Soldier>().isFollowing = false;
            actionManager.Stop();
            for (int i = 0; i < 9; ++i) {
                soldiers[i].GetComponent<Animator>().SetFloat("Speed", 0f, 0.1f, Time.deltaTime);
            }
        }
    }
}

// 巡逻兵碰撞玩家
private void OnSoldierCollideWithPlayer() {
    StartCoroutine(Dead());
    player.GetComponent<Rigidbody>().isKinematic = true;
    view.state = model.state = GameState.LOSE;
    soldiers[currentArea].GetComponent<Soldier>().isFollowing = false;
    soldiers[currentArea].GetComponent<Rigidbody>().isKinematic = true;
    for (int i = 0; i < 9; ++i) {
        soldiers[i].GetComponent<Animator>().SetFloat("Speed", 0f, 0.1f, Time.deltaTime);
    }
    actionManager.Stop();
    Cursor.lockState = CursorLockMode.Confined;
}

动作分离

在 MoveToAction 类中实现巡逻兵的自主巡逻，在 TraceAction 类中，实现了巡逻兵追赶玩家，随后利用 ActionManager 对动作进行管理，每当相应的事件发布/订阅时，进行动作的处理：

public class MoveToAction : Action {
  public Vector3 target;
  public float speed;
  public int area;
  public static MoveToAction GetAction(GameObject gameObject, IActionCallback callback, Vector3 target, float speed, int area) {
    MoveToAction action = CreateInstance<MoveToAction>();
    action.gameObject = gameObject;
    action.transform = gameObject.transform;
    action.callback = callback;
    action.target = target;
    action.speed = speed;
    action.area = area;
    return action;
  }
  public override void Start() {
    if (target - gameObject.transform.position == Vector3.zero) { return; }
    
    Quaternion rotation = Quaternion.LookRotation(target - gameObject.transform.position, Vector3.up);
    gameObject.transform.rotation = rotation;
  }
  public override void Update() {
    if ((gameObject.transform.position - target).magnitude < 0.1f) {
      destroy = true;
      callback.ActionDone(this);
    } else {
        // 使用 MoveTowards 将巡逻兵移动到指定位置
      gameObject.transform.position = Vector3.MoveTowards(gameObject.transform.position, target, speed * Time.deltaTime);
    }
  }
}

public class TraceAction : Action {
  public GameObject target; 
  public float speed;
  public static TraceAction GetAction(GameObject gameObject, IActionCallback callback, GameObject target, float speed) {
    TraceAction action = CreateInstance<TraceAction>();
    action.gameObject = gameObject;
    action.transform = gameObject.transform;
    action.callback = callback;
    action.target = target;
    action.speed = speed;
    return action;
  }
  public override void Start() { }
  public override void Update() {
    gameObject.transform.position = Vector3.MoveTowards(gameObject.transform.position, target.transform.position, 1.5f * speed * Time.deltaTime);
    if (gameObject.GetComponent<Soldier>().isFollowing == false || (gameObject.transform.position - target.transform.position).sqrMagnitude < 0.00001f) {
      destroy = true;
      callback.ActionDone(this);
    } else {
        // 使用 Quaternion 四元数对玩家位置定位，进行球面插值获得位置差距
      Quaternion rotation = Quaternion.LookRotation(target.transform.position - gameObject.transform.position, Vector3.up);
      gameObject.transform.rotation = rotation;
    }
  }
}

项目心得

本次项目加强了我对使用 Unity 进行角色动画处理，以及刚体和碰撞的理解，同时如何将用户的输入与动画进行挂钩，如何处理动画并且使用 Blend Tree 将动画组合，实现不同形态的动画转移。

同时，我发现有时候动画会出现延迟的问题，经过查阅资料后，我发现是因为有些动画需要退出时间，需要在相应的动画 Transition 上取消 Has Exit Time，就会连贯很多：

除此之外，我也能更好地利用面向对象编程，进行游戏设计，将动作分离并且利用订阅/发布模式处理相应的事件，尽管本次项目中有些地方没有处理地很好，比如有时帧率不太高，角色动画不连贯，碰撞体积不对等等，但总的来说，收获了很多。
