Javascript in JSX With Curly Braces

JSX 允许你在 Javascript 中编写类似 HTML 标记语言的标签，将渲染逻辑和内容保存在同一个文件中。 有时你想增加一点 Javascript 逻辑或是在标记语言中引用一个动态变量，你可以在 JSX 中使用花括号，通过打开一个窗口通向 Javascript (open a window to Javascript, 找不到信达雅的翻译)

在本章节你将会学到：

  • 如何使用引号传递字符串
  • 如何在 JSX 中使用花括号引用一个 Javascript 变量
  • 如何在 JSX 中使用花括号调用一个 Javascript 函数
  • 如何在 JSX 中使用花括号获取一个 Javascript 对象

Where to use curly braces

你只能在 JSX 中按照如下两种方式使用花括号：

  1. 在 JSX 标签内部作为文本内容使用： <h1>{name}'s To Do List</h1> 正常工作，但 <{tag}>Gregorio Y. Zara's To Do List</{tag}> 则不会正常工作
  2. 在 JSX 标签中作为属性传递，跟在 = 后面：src={avatar} 将会读取 avatar 变量，但 src="{avatar}" 会传递字符串 {avatar}

Using “double curlies”: CSS and other objects in JSX

除了传递字符串，数字和其他 Javascript 表达式以外，你还可以在 JSX 中传递对象。由于对象同样使用花括号表示，比如 { name: 'Hedy Lamarr', inventions: 5}。因此，为了传递一个 JS 对象，你必须再使用一个花括号来将其包起来：person={{ name: "Hedy Lamarr", inventions: 5 }}

你可能需要在 JSX 中使用内联 CSS，但在 React 中这并不是必须的，因为大部分场景下，类选择器也能很好地工作。

总地来说，在 JSX 中如果遇到了两个花括号 {{ 和 }}，不用担心，它本质其实就是一个在花括号中的对象！

陷阱：

内联 CSS 属性使用驼峰表示法 camelCase

比如 HTML 中的 <ul style="background-color: black">，

在 JSX 中的表示为 <ul style={{ backgroundColor: 'black' }}>

More fun with JavaScript objects and curly braces

你可以将多个表达式放到一个对象中，然后在 JSX 中使用花括号引用他们：

在这个例子中，person 对象具有 name 字符串属性和 theme 对象：

const person = {
  name: "Gregorio Y. Zara",
  theme: {
    backgroundColor: "black",
    color: "pink",
  },
};

在组件中可以点成员访问 person 的成员：

<div style={person.theme}>
  <h1>{person.name}'s Todos</h1>

JSX 由于可以使用 Javascript 组织数据和逻辑，作为一种模板语言是非常 minimal 极小化的

Recap

回顾本章，你已经学到 JSX 中大部分语法了：

  • 使用引号来传递 JSX 字符串属性
  • 使用花括号在标记语言中访问 Javascript 的逻辑和变量
  • 花括号在 JSX 标记语言标签内和属性等号后作用
  • {{ 和 }} 双重大括号并不是特殊语法，而是 Javascript 中的对象，被塞入了 JSX 花括号中
