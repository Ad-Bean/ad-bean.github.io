写在前面：

最近 React 正式发布 React18，于是想尝试捡起博客，翻译一下 Beta 文档

但本系列并非是全文机翻，会存在很多本人的主观理解，如有错误，还请原谅和指出

React docs

测试版本的 React 文档主要分为两部分：

  • Learn React：作为学习 React 的主要教程，旨在使用更多可互动的例子教会阅读者如何编写 React，更重要的是 React 思维即 Think in React
  • API Reference：API 接口文档，这部分完成度不是很高，但样例非常多。

为什么要有新的文档

既然已经有 React 官方文档，为什么还要有一个 beta 版本的文档呢？

  • beta 文档所有样例使用 Hooks 钩子，而不是使用 class
  • 更多的可交互的样例，和可视化的图表
  • 更多包含解决办法的挑战，帮助你更好地理解和阅读文档

但 beta 文档仍在开发当中，目前 Learn React 学习部分已完成 70% 左右，而 API 接口部分只有 5%

题外话：

根据 github 上的讨论，beta 文档的工作实际上早在 2020 年就已经开始，但由于各种原因，疫情影响、圣诞假期、团队规模较小、正式发布新版本 18 等等任务占据了大部分工作，React beta 文档的更新工作已经暂缓了很久。

所以 beta 文档内容并不能涵盖所有使用 React 的场景，尤其是 useEffect 部分，更全面的内容还是需要回到 reactjs.org 和搜索其他人的文章去理解，后续我也会根据他人的文档来补充这一方面。

如何提供反馈

如题外话中所说，如果你对于 beta 文档中的任何地方存在疑问，可以到 Github issue 中反馈，或是匿名信。除此之外，文档每一页都有一个反馈按钮，如果你在阅读文档时，发现一些不合理的地方，可以及时反馈

Beta 文档会最终替代主页的文档吗

React 团队的目标就是，一旦 beta 文档的内容和官方主页文档一样丰富时，就替换掉主页文档，使主页文档成为一个子文档，依然可以阅读，但会注明那些内容已过时。

结语：

正如题外话所说，虽然 beta 文档的立意是好的，使用 function component 函数组件，以及 hooks 来取代 class component 的写法，但该文档的更新速度在一定程度上拖慢了这个进程。
