Your First Component 你的第一个组件

组件 Components 是 React 的核心概念之一，组件是构建用户界面 UI 的基础。学习如何创建组件将是学习 React 一个很好的开始。

在本章节你将会学到：

  • 什么是组件
  • 组件在 React 应用中的角色
  • 如何编写 React 组件

组件：组成 UI 的积木

原标题 Components: UI building blocks，不知道如何优雅翻译

在 Web 应用/网页中，HTML 标记语言允许我们使用一系列内置标签创建富文本结构，如 <h1> 和 <li>

<article>
  <h1>My First Component</h1>
  <ol>
    <li>Components: UI Building Blocks</li>
    <li>Defining a Component</li>
    <li>Using a Component</li>
  </ol>
</article>

上述标记语言表示一个文本 <article>，使用标题标签 <h1> 和一个有序表格（缩写标签 ordered list）标签 <ol>。像这样的标记语言，结合 CSS 编写样式，以及 Javascript 实现交互性，就是你在 Web 上看到的所有 UI，不论是侧边栏，还是头像、模型或是下拉菜单。

React 允许你在一个自定义的组件中，结合 HTML 标记语言 CSS 样式以及 Javascript 语言，形成可以在应用程序中复用的 UI 组件元素。比如上述的表格 table 中的内容可以转换成一个 <TableOfContents /> 组件，你可以在每个页面都渲染这个表格。Under the hood 本质来说，这其实就是 HTML 中的标签，比如 <article>, <h1>。

和 HTML 标签一样，你可以自由组合、排序和嵌套各种组件来设计整个页面。举个例子，你当前正在阅读的文档就是 React 组件构成的：

<PageLayout>
  <NavigationHeader>
    <SearchBar />
    <Link to="/docs">Docs</Link>
  </NavigationHeader>
  <Sidebar />
  <PageContent>
    <TableOfContents />
    <DocumentationText />
  </PageContent>
</PageLayout>

当你的项目开始增长，你会注意到许多设计是可以将其编写成组件，以便后续重用和复用，从而加快开发过程。上述的表单 <TableOfContents /> 就可以添加到任何地方，你也可以使用许多 React 开源社区中的组件库比如 Chakra UI 和 Material UI

定义一个组件

当创建一个传统的网页时，网页开发者一般使用标记语言填充内容，然后使用 Javascript 为网页添加交互功能。当网页需要互动的时候，这样的流程是可以接受的。但现在，很多网站或应用程序需要更多。React 将交互性放在首位的同时，使用了同样的技术栈，即：一个 React 组件就是一个 Javascript 函数，你可以在这个函数中使用标记语言。下面是一个例子：

Step 1：导出组件

export default 前缀是一个标准的 Javascript 语法（不是 React 特有的），它允许你在文件中标记处主要的函数，以便后续可以在其他文件中导入该函数（更多内容请查看导出和导入组件章节）

Step 2：定义函数

使用 function Profile() { } 定义一个 Javascript 函数，组件名称为 Profile

陷阱：

React 组件就是常规的 Javascript 函数，但首字母必须大写，否则不会起作用

Step 3：添加标记语言

这个组件返回一个 <img /> 标签，带有 src 和 alt 属性，就像 HTML 一样，但他的本质上又是 Javascript。这种语法被称为 JSX，它可以让你在 Javascript 中嵌入标记语言。

返回语句可以只写一行，比如：

return <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />;

但如果你的标记语言不止一行，即和 return 关键字不在同一行，则必须使用括号将其包起来：

return (
  <div>
    <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
  </div>
);

陷阱：

如果没有括号，所有 return 后面的代码将会被忽略

原因就是 Javascript 中自动分号的策略，详情查看 stackoverflow 的回答

使用组件

现在你已经定义了 Profile 组件，你可以将其嵌套到其他组件中使用，比如你可以导出一个嵌套多个 Profile 信息组件的 Gallery 画廊组件，

浏览器看到的

可以发现 React 组件中的标记语言和 HTML 是有差异的：

  • <section> 为小写，所以 React 知道这是 HTML 标签
  • <Profile /> 首字母大写，所以 React 知道需要使用 Profile 组件而不是 HTML 标签

并且 Profile 组件比 HTML 标签拥有更多内容，比如 <img/>，所以浏览器看的是其实是：

<section>
  <h1>Amazing scientists</h1>
  <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
  <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
  <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
</section>

嵌套和组织组件

正如我们之前所说，React 组件实际上就是 Javascript 函数，所以你可以在一个文件中实现多个组件。这很方便，尤其是当组件都比较小，而且都紧密联系时，但一旦当文件开始变得拥挤，你可以将 Profile 组件单独开一个文件。下一篇文章，即导入和导出组件会仔细讨论这个问题。

因为 Profile 组件在 Gallery 组件中多次渲染，即画廊组件 Gallery 是一个父组件，而每个在父组件里渲染的 Profile 组件被称为子组件。

这就是 React 的魔力之处，你可以一次声明定义一个组件，却可以在任何地方任意时间多次使用。

深入了解：Components All the Way Down

你的 React 应用应该从根组件 root 开始。通常，当你创建一个新 React 项目时会自动创建一个根组件。比如，当你使用 CodeSandbox 或 Creat React App 时，根组件就是 src/App.js。如果你使用其他框架比如 Next.js，则根组件是 pages/index.js。在这些例子中，已经自动导出了根组件。

大部分 React 应用自始至终都使用组件，即 All the way down。这意味着，你不仅要使用可重用的小组件，比如按钮，还需要使用更大的组件比如侧边栏、列表甚至是整个页面。组件是组织 UI 代码和标记语言的便捷方式，哪怕其中一些组件只使用一次。

一些框架比如 Next.js 则更进一步，不是使用一个单独的 HTML 文件和让 React 使用 Javascript 掌管整个页面，Next.js 是将 React 组件自动转换成 HTML，即允许在 Javascript 加载之前展示一些内容。

尽管如此，许多网站只使用 React 来增加可交互性。他们有许多根组件，而不是一个。你可以任意使用 React，全部或者很少。

回顾

通过这篇文章，我们对 React 有了一个初步的了解，回顾这篇文章：

  • React 允许你为你的应用创建组件，可复用的 UI 元素。
  • 在一个 React 应用中，用户界面 UI 中的任意元素都可以是组件。
  • React 组件实际上就是 Javascript 函数，但有以下不同：
    • React 组件通常以首字母大写开头
    • React 组件返回的是 JSX 标记语言

挑战 1/4：正确导出组件

React 的编译报错还是很友善的，会提醒你是否是忘了导出组件，解法在第一行

挑战 2/4：正确返回 JSX 语句

React 编译报错提醒你该组件什么都没有返回，添加括号或将其放在同一行，并且要记得删掉 HTML 标签后面的分号

挑战 3/4：找出错误

这样的错误 React 并不会编译报错，所以需要记住组件一定是首字母大写

挑战 4/4：编写你的组件

任意编写一个 React 组件即可，注意前面三个问题即可，正确导出，正确返回，首字母大写
