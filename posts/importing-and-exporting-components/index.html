Importing and Exporting Components 导入和导出组件

组件的魅力之处在于它们是可重用的，你可以创建由其他组件组成的新组件。但当你嵌套足够多的组件时，你可能需要将不同组件拆分成不同的单独的文件。这样可以使得你的文件结构更清晰，容易找到组件以及在更多地方复用组件。

在本章你会学习到

  • 什么是根组件
  • 如何导出和导入组件
  • 什么时候使用默认 导出 default 和命名 named 导出和导入
  • 如何在一个文件导入导出多个组件
  • 如何拆分不同的组件到不同的文件

根组件

在你的第一个组件 Your First Component一章中，你已经制作了一个 Profile 组件和一个 Gallery 组件，并且可以成功渲染它们：

这些组件目前存在一个根组件文件中，即 App.js。在 Create React App 一章中，你的 React 根组件就是在 src/App.js 文件中。根据你的环境，根组件有可能在别的文件中，如果你使用了带路由的框架比如 Next.js，则每个页面的根组件可能会不同。

导入和导出组件

如果你想要改变 landing screen（不知道怎么翻译，可能叫首屏比较好），并且想放一些科学书籍的清单在首屏，又或者你想将 Profile 放到别的地方。显而易见地，你会想到将 Gallery 和 Profile 组件移出根组件文件。这使得这些组件更加模块化和可重用，你可以通过以下三个步骤来移动组件：

  1. 创建一个新的 JS 文件，将组件放入文件中。
  2. 在该文件中导出 Export 你的函数组件（使用 default 或者 named 两种方式导出，即默认导出（每个模块包含一个）、命名导出（每个模块包含任意数量））
  3. 在你需要使用这些组件的文件中导入组件（使用相应的默认导入、或命名导入）

在本例，我们将 Profile 组件和 Gallery 组件移出根组件 App.js，并且创建相应的 JS 文件 Gallery.js，现在你可以在根组件 App.js 中将其导入。

仔细观察这个例子是如何将组件拆分成两个文件的：

  1. Gallery.js
    • 在同一个文件下定义 Profile 组件，只用于该文件，并不会被导出
    • 使用默认导出 default export 导出 Gallery 组件
  2. App.js
    • 使用默认导入从 Gallery.js 文件中导入Gallery 组件
    • 使用默认导出 default export 导出 App 根组件

注释

你可能会遇到不带 .js 后缀的文件导入比如 import Gallery from './Gallery'

不论是 import Gallery from './Gallery.js' 还是 import Gallery from './Gallery'，在 React 中都可以正常工作。尽管前者更接近 native ES Modules 的工作原理，即 JS 中的模块

更多知识：默认导出和命名导出

在 Javascript 中有两种主要的方法导出值，默认导出和命名导出，即 default exports 和 named exports。目前为止，我们的例子都只使用了默认导出。但你可以在同一个文件中使用两者中的任意一种，但一个文件只能有一次默认导出，但可以有多个命名导出。

如何导出你的组件决定了你该如何导入这个组件，如果你尝试使用命名导入一个默认导出的组件，则会导致报错。下面的表格会帮助你如何正确导入导出：

当你使用了默认导入，你可以在导入 import 后面将组件重命名为任何你想要的名字。比如你可以 import Banna from ./Button 而且他依然会默认导出 Button 组件。相反地，对于命名导入，你一定要匹配导入和导出的名字，这也是为什么它被称为命名导入。

如果文件只有一个组件需要导出，人们常常使用默认导出，而对于多个组件、值需要导出则使用命名导出。但不论你喜欢那种编码风格，请始终为你的函数组件和文件起一个有意义的名称。没有名字的组件，比如匿名函数 export default ()=>{} 是不被鼓励的，因为它们使得调试更加困难。

在同一个文件中导出和导入多个组件

如果你只是想站是一个 Profile 而不是一整个 Gallery，你也可以导出 Profile 组件。但 Gallery.js 已经有了一个默认导出即 default export，你不能在一个文件中有两个默认导出。你可以创建一个新的文件再使用默认导出，也可以使用命名导出 Profile 组件。一个文件只可以有一个默认导出，但可以拥有很多命名导出。

为了减少默认导出和命名导出之间存在的潜在混淆，一些团队选择只用一种导出风格（默认或命名导出），或者避免将他们混在一个文件中

当然这是个人喜好问题，选择你喜欢的方式即可。

首先，在 Gallery.js 文件中导出 Profile 组件，使用命名导出（即没有 default 关键字）

export function Profile() {
  // ...
}

然后在 App.js 根组件中，从 Gallery.js 文件使用命名导入 Profile 组件（使用花括号命名导入）

import { Profile } from './Gallery.js';

最后，在 App 根组件中渲染 <Profile /> 组件

export default function App() {
  return <Profile />;
}

现在 Gallery.js 文件含有两个导出，一个默认导出 Gallery 和一个命名导出 Profile。而在根组件 App.js 文件中有对应两个导入，在原文章中可以尝试修改该样例：

对本例子，使用了混合的默认导入导出和命名导入导出：

  • Gallery.js:
    • 使用命名导出 Profile 组件，且命名就是 Profile
    • 使用默认导出 Gallery 组件
  • App.js:
    • 使用命名导入 Profile 组件，且命名为 Profile（注意有花括号）
    • 使用默认导入 Gallery 组件（默认导入没有花括号）
    • 使用默认导出，导出根组件 App
