PC 寄存器

程序计数器 program counter 保存着下一条指令的地址，并且当该条指令，解码执行完成后，会更新为下一条指令的地址。

IR 寄存器

指令寄存器 instruction register 保存当前正在执行或解码的指令，并且在指令被解码、准备和最终执行的过程中保存着指令。

ACC 寄存器

累加器 accumulator 用来储存计算产生的中间结果，如果没有像累加器这样的寄存器，那么在每次计算 (加法，乘法，移位等等) 后就必须要把结果写回到内存，也许马上就得读回来。然而存取主存的速度是比从算术逻辑单元到有直接路径的累加器存取更慢。

Fetch-Execute 周期

LOD #3

Fetch 获取：从程序计数器 PC 中取得该指令的的地址码，并且将该指令送入指令寄存器 IR

Decode 解码：指令由处理器解码，并且将结果传给 MUX 多路选择器，将不同的操作数输入传给 ALU 算术逻辑单元，并且控制 ALU 算术逻辑单元的功能。

Execute 执行：在最后一个阶段，处理器执行指令。ALU 算术逻辑单元将操作数 #3 从指令中指定的地址拷贝到 ACC 累加器寄存器中。

ADD W

Fetch 获取：从程序计数器 PC 中取得该指令的的地址码，并且将该指令送入指令寄存器 IR

Decode 解码：指令由处理器解码，并且将结果传给 MUX 多路选择器，将不同的操作数输入传给 ALU 算术逻辑单元，并且控制 ALU 算术逻辑单元的功能。

Execute 执行：在最后一个阶段，处理器执行指令。处理器从内存 W 中取出操作数 3，与累加器储存器中的操作数 7 相加，ALU 算术逻辑单元将相加后的结果储存到 ACC 累加器寄存器中。

寻址方式

“LOD #3” 与 “ADD W” 指令的执行在 Fetch-Execute 周期级别中主要是寻址方式不同。

LOD 是立即数寻址，即从指令中得到 #3 作为操作数，而 ADD W 则要从寄存器 W 中获得相应的数据作为操作数，再与累加寄存器中的操作数相加。

其次，LOD 的指令格式与 ADD 也不相同。

机器码

LOD #7 的二进制形式：00010100 00000111

参照 [The PIPPIN User’s Guide]

Opcode Operand(Binary) Assembly Instruction Description
00000100 bbbbbbbb LOD X Load the accumulator with the 8-bit value stored at referenced memory address.
00010100 bbbbbbbb LOD #n Load the accumulator with an immediated value.

其中前八位指令代表 LOD，第四位为 0 则从寄存器读取操作数，第四位为 1 则是读取一个立即数

后八位则为寄存器地址或立即数的二进制形式，在这里$ (00000111)_ 2 = 7_{10} $

RAM 地址

该 CPU 的 RAM 从 0 - 110，二进制为 00000000 到 01101110，用于存储指令。其中每条指令的长度为 16 位 2 个字节。 而数据存在从 10000000 到 10000111 的寄存器中，其中每个寄存器大小为 8 位。

CPU 位数

从图上可知该 CPU 的 ACC 寄存器可以储存 8 位大小的数据，所以该 CPU 是 8 位 CPU。

机器语言和高级语言的区别与联系

通过模拟 CPU，我直观地感受到汇编语言、机器语言是对应着不同的机器语言指令集，汇编语言通过汇编过程转换成机器指令。特定的汇编语言和特定的机器语言指令集是一一对应的，是一种直接面向硬件的语言，但是对于递归、循环等等操作，相较于高级语言代码量大且复杂，不便于编写和维护。

高级语言语法和结构更类似人类语言，更加简单、易于理解和维护。高级语言经过编译后可以得到目标程序，高级语言要通过编译程序翻译成汇编代码，汇编语言要通过汇编得到机器语言。

虽然他们之间存在区别，但是联系还是非常紧密的。
