创建和嵌套组件 Components

React 应用是由组件组成的，一个组件是一小块 UI（User Interface）用户界面的一部分，拥有自己的逻辑和外观。组件可以像按钮一样小，也可以是包含整个页面。

React 组件是返回 Markup 标记语言的 Javascript 函数：

function MyButton() {
  return (
    <button>Click me</button>
  );
}

这样你就已经声明了 MyButton 组件，你可以将整个组件嵌入到另一个组件中。

export default function MyApp() {
  return (
    <div>
      <h1>Welcome to my app</h1>
      <MyButton />
    </div>
  );
}

注意到 MyButton 以大写字母开头，这样你就可以知道它是一个 React 组件。React 组件必须始终以大写字母开头，而 HTML 标记语言则是小写的。

使用 JSX 编写 markup 标记语言

上面的 markup 标记语法使用的是 JSX，但 React 并不一定必须使用 JSX。JSX 是可选的，但大多数时候，使用 JSX 可以很方便地编写标记语言。

JSX 要比 HTML 标记语言更加严格。你必须使用闭合标签比如 <br />。组件不能返回简单的 JSX 标签，而是必须将其包含在一个共同的父亲组件中，比如 <div> ... </div> 或是一个简单的空元素 <> ... </>。

function AboutPage() {
  return (
    <>
      <h1>About</h1>
      <p>Hello there.<br />How do you do?</p>
    </>
  );
}

如果你需要将 HTML 转换成 JSX 语法，可以使用在线转换器

添加样式

在 React 中，CSS 中的类名需要使用 className 来添加，与 HTML 中的类属性是一致的，但使用了驼峰命名。

<img className="avatar" />

然后你就可以在分离的 CSS 文件中编写 CSS 规则了：

/* In your CSS */
.avatar {
  border-radius: 50%;
}

React 并没有规定如何添加 CSS 文件。在简单的情况下，你可以在 HTML 文件中添加一个 <link> 标签。如果你使用的是框架或构建工具，则需要参考其文档如何添加 CSS 文件。

这一部分还需要补充，但由于本人在使用 React 时习惯使用 chakra-ui 或 tailwind-css 等 ui 库，即直接通过添加类名来添加样式，所以后续再来补充。

展示数据

JSX 可以让你在 Javascript 中使用标记语言，使用花括号可以让你 “escape back” 逃回 Javascript 使得你可以嵌入一些变量到你的代码里，然后展示给用户，比如：

return (
  <h1>
    {user.name}
  </h1>
);

反过来你可以 “escape into Javascript” 逃入 Javascript，同样使用花括号而不是引号，比如 className="avatar" 只是将字符串传入了 CSS 的类名，而 src={user.imageUrl} 则是通过读取 Javascript 对象中的 user.imageUrl 变量字段，传入给 src 属性：

return (
  <img
    className="avatar"
    src={user.imageUrl}
  />
);

条件渲染

Conditional rendering，在 React 中，并没有针对条件表达式的特殊的语法。相反，你可以使用常规的 Javascript 代码，比如 if 条件语句：

let content;
if (isLoggedIn) {
  content = <AdminPanel />;
} else {
  content = <LoginForm />;
}
return (
  <div>
    {content}
  </div>
);

如果你更喜欢紧凑的代码，可以使用条件运算符（三目运算符）?:

<div>
  {isLoggedIn ? <AdminPanel /> : <LoginForm />}
</div>

如果你并不需要 else 分支，可以使用逻辑运算符，与 &&

<div>
  {isLoggedIn && <AdminPanel />}
</div>

所有这些方法同样可以使用在属性上，比如 className={isBlack ? 'black' : 'white'}

渲染列表

渲染列表是一个很有用的功能，他依赖于 Javascript 中 for, map, forEach, ... 等循环功能，比如我们有如下一个数组：

const products = [
  { title: 'Cabbage', id: 1 },
  { title: 'Garlic', id: 2 },
  { title: 'Apple', id: 3 },
];

在你的组件内部，使用 map() 函数将数组转换为 <li> 数组：

const listItems = products.map(product =>
  <li key={product.id}>
    {product.title}
  </li>
);

return (
  <ul>{listItems}</ul>
);

注意到 <li> 有一个 key 属性，对于列表中的每一项，应该传入一个字符串或数字，作为该项的唯一标记。通常，key 应该来自于数据本身，比如数据库中的 ID，React 使用该 key 判断 DOM 发生了什么改变，比如插入、删除或重新排序。

事件响应

你可以在组件中通过声明事件处理函数 event handler functions 来响应事件，

function MyButton() {
  function handleClick() {
    alert('You clicked me!');
  }

  return (
    <button onClick={handleClick}>
      Click me
    </button>
  );
}

注意到 onClick={handleClick} 仅仅传入了函数的名称而没有括号，即不会调用函数，而是传参。React 会在当你按下按钮时调用事件处理按钮。

更新屏幕

题外话：

我觉得这一个标题起的不是很好，什么叫更新屏幕呢？是更新 DOM 吗？

通常，你会希望你的组件能记住一些信息并且展示它们。比如，你会想要实现一个计数器，每次按下按钮时，计数加一。

为了实现计数器，你需要使用 state 即状态，首先引入 useState 钩子即 hooks：

import { useState } from 'react';

function MyButton() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <button onClick={handleClick}>
      Clicked {count} times
    </button>
  );
}

随后在组件中声明一个状态变量，即 [count, setCount] = useState(0)，可以发现 useState 返回两个变量，前者是当前的状态 count，而后者是一个函数 setCount，用于更新状态。

当第一次展示按钮时，count 的值为 0，这是因为你在 useState(0) 传入了 0，用于初始化。当你想更改状态时，调用 setCount()，传入新的状态，更新状态，比如上述的计数器。

当你按下按钮，React 会再次调用组件函数，此时 count 会变成 1，然后再按下，再调用，更新状态，渲染等等….

如果你有多个同样的组件，此时，每个组件都有自己的状态，如下，分别点击不同计数器的计数按钮：

此时每个按钮都有自己的状态，并不会相互影响。

组件之间共享状态

上文中每个 MyButton 组件都有自己独立的状态，当按钮被点击时，该组件的 count 状态单独改变：

但是，你经常需要组件共享数据，并且这些数据需要一同更新。

为了使得两个 MyButton 组件显示相同的 count 状态，并一起更新，需要进行状态提升，将状态从单个组件中提升到最近的公共组件，比如：

现在，当你点击按钮时，count 状态由于是在 MyApp 中，通过更新后再传入给子组件 MyButton，会使得其一同改变。

下面是代码：

function MyButton({ count, onClick }) {
  return (
    <button onClick={onClick}>
      Clicked {count} times
    </button>
  );
}

export default function MyApp() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <h1>Counters that update together</h1>
      <MyButton count={count} onClick={handleClick} />
      <MyButton count={count} onClick={handleClick} />
    </div>
  );
}

整个流程：

  • 状态提升，从 MyButton 到 MyApp
  • 将 MyApp 中的状态 state，向下传递给子组件 MyButton，这里同样将事件处理函数一同传递
  • 向下传递的状态又称 props 参数
  • 最后修改 MyButton 中的参数，读取传入的 props，比如传入的 onClick 事件处理函数，当发生点击事件后，onClick 触发，每个按钮的 onClick 会被设为父组件中的 handleClick 并且会设置 setCount(count + 1) 更新状态变量，并且新的状态又会通过 props 传给每个子组件，展示新的状态。

这就是所谓的 ‘lifting state up’ 即状态提升，通过提升状态，我们在组件之间共享状态。
