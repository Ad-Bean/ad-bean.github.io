Describing the UI 描述用户接口

React 是一个用于渲染用户界面 UI 的 Javascript 库。用户界面 UI 是由按钮、文本、图像等小单元构建的。React 可以让你组合这些小单元成为一个可重用、可嵌套 的组件。不论是网页还是手机应用，所有在屏幕上展示的都可以被拆解为组件。

在这一章节，你将学到如何去创建、自定义和根据条件展示 React 组件。

本章是一个概要，下面的小标题是每一章的主要内容，更深入的内容需要阅读每一章节的全部内容

在 Javascript 和 JSX 使用花括号

JSX 允许你可以在 Javascript 中编写类似 HTML 的标记语言，并且在同一个位置渲染逻辑和内容。有时候，如果你想在 Javascript 中加一些 JS 逻辑或在标记语言中引用一个动态属性，这种情况下，你可以使用 JSX 中的花括号，即在 Javascript 中开一个新窗口 “open a window”。

渲染列表

通常，你可能希望从一个数据集合中展示多个相似的组件。你可以在 React 中使用 Javascript 中的 filter() 和 map() 函数处理数组，过滤或转换数组中的数据，将其组成一个组件。

对于数组中的每个元素，你还需要指定它们的 key 关键字。通常，你可以通过数据在数据库中的主键 ID 来指定 key。Keys 使得 React 可以很好地跟踪每个元素在数组中的位置，监听其变化。

有一些 Javascript 的函数是纯函数，即 “pure” 的：

  • Minds its own business. 纯函数在被调用之前，不会改变已存在的任何对象或变量。
  • Same inputs, same output. 给定相同的输入，纯函数总是返回相同的结果。

你编写的组件应该是严格的纯函数，这样你可以避免随着代码量的增长，出现一些令人困惑的 bug 和不可预知的行为，下面是非纯函数的组件例子：
